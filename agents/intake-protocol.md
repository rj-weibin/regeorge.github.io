# 外部资料摄入协议

## 目标

让外部资料进入仓库时有隔离区、有来源、有碰撞检索和人工确认，不把临时搬运内容直接变成事实。

## 流程

1. 将外部资料放入 `inbox/pending/`，记录原始来源和摄入日期。
2. 识别资料类型：书籍、原话、洞见、项目、工作流、技能、决策或资源。
3. 按标题、关键词、稳定 ID 和 links 检索 `data/`，确认是否已有实体。
4. 输出最小碰撞结果：相关已有实体、用户原话、可能的新洞见、需要用户决定的动作。
5. 等用户确认“补充已有”“新建洞见”“仅保留记录”或“忽略”。
6. 只将确认后的内容写入 `data/`，补齐 Frontmatter 和关联。
7. 运行 `node scripts/validate-data.mjs`，再运行 `node scripts/build-site.mjs`。

## 禁止

- 不直接写 `pages/`。
- 不把书的观点伪装成用户洞见。
- 不因表达相似就自动合并实体。
- 不把整份外部资料复制到多个分类目录。
