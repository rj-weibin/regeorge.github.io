# 阅读碰撞协议

当用户同时提供书名和读后感、摘录或明确的阅读感受时触发；只有书名或“只是记录读过”不自动碰撞。

## 识别

先在 `data/books/` 查找书籍实体，再查找其 `links`、概念 ID 和历史碰撞。找不到书籍时只建立 `inbox` 草案，不凭空补齐书籍事实。

## 输出

输出不超过必要范围的碰撞摘要：

- 已有洞见：引用稳定 ID 和原有原话；
- 书中触发点：只记录用户提供或明确指出的内容；
- 用户原话：一字不改；
- 候选洞见：区分“已有补充”和“可能新建”；
- 裁决选项：补充已有、升级新卡、仅保留阅读记录、跳过。

不要写书评，不替用户总结整本书，不把表达差异判为张力。

## 写入门禁

用户确认前保持 `proposed`，不得写入 `reviewed` 实体。确认后再更新书籍与概念之间的双向关系，并将来源写入 `source`。
